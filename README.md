# client-server-security-lab
Built a client-server lab to analyze secure communication and web vulnerabilities

## Objectives
- Understand how client-server communication works
- Analyze HTTP vs HTTPS traffic
- Identify potential vulnerabilities in web communication

## Tools Used
- Linux
- Apache Web Server
- SSH
- Google Cloud Platform (GCP)

## What I Did
- Deployed a web server in a virtual environment
- Configured networking and access controls
- Intercepted and analyzed traffic using Burp Suite
- Compared secure vs non-secure communication (HTTP vs HTTPS)

## Key Takeaways
- Gained hands-on experience with web traffic analysis
- Learned how vulnerabilities can be identified in client-server communication
- Strengthened understanding of web security fundamentals
